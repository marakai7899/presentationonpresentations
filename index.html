OPEN SOURCE PRESENTATIONS

USING WEB FRAMEWORKS

WHY USE WEB FRAMEWORKS?

REUSABILITY
Content including markup, stylesheets, scripts and images can easily be copied to blog posts, web pages and presentations

UBIQUITY
Files can be stored on Github and accessed on any device with an internet connection at any time.

SCALABILITY
API's can be used to add all kinds of cool features, such as real time updates or content on the fly

EFFICIENCY
Web frameworks reduce the time spent on formatting images and fonts. Moreover, they do not require any software other than a web browser.

RESPONSIVENESS
Presentations can be accessed on any device without compromising usability or aesthetics

ATTRACTIVENESS
Attractive, full screen images can be used and made responsive across different devices.

REUSABILITY

  • Content can be reused across different mediums

    This includes blogs, newsletters, web pages and more

  • No need to download fonts

    Unlike MS Powerpoint fonts do not need to be downloaded on each individual device in order to be rendered correctly. They can be served over the internet.

  • Stylesheets can be served from the internet

    Stylesheets can also be served from the internet so there is no need to copy color codes and other styles.

  • No need to format headings

    No need to format headings as the copied content will already be formatted

  • Semantic elements like tables are pre-formatted

    Semantic elements like tables, paragraphs, captions and other elements are pre-defined and formatted

  • In most cases, images do not need to be resized

DON'T BELIEVE ME?

The process of transferring the list from the presentation to the write-up on my website took only a few seconds with no need to format the content.

UBIQUITY

  • Projects are available 24x7

    Projects are available on the internet through GitHub or other web hosts

  • Track changes

    If users choose to host on GitHub they can track changes and access previous versions

  • Not platform-dependent

    Nearly every web browser can access them so they are not dependant on a particular operating system or platform

  • Projects can look nearly uniform across devices

    Users can ensure that properly coded projects will look nearly uniform across multiple devices and multiple browsers

  • No need for on premise hard drives or other storage devices
  • Access most recent version of the project Presenters can make sure that they are accessing the most recent version of the presentation

SCALABILITY

  • Presentations need to grow and evolve with emerging technologies

    The web is going to get disrupted very soon by the arrival of artificial intelligence, internet of things, virtual reality and more. Presentations need to grow and evolve with these developments

  • Use interactive featuresincluding VR and live social media updates

    Very few presentations, even the ones that present on technology, use virtual reality or high degrees of user engagement. Web frameworks which can integrate with nearly all technology can be scaled up to achieve features like displaying live tweets.

  • Technology extends to fit user needs

    I love Wordpress. Everybody should love Wordpress. There is in fact an option to use a Wordpress theme built with RevealJS by the team at Alley Interactive. I am optimistic that in time more plugins and related frameworks will be developed.

  • Nearly all third party integrations are compatible

    There is no limit on the number of third party integrations or functionality that can be added as it is only a matter of scaling up a web host plan.

  • Increased use will incentivise developers to develop this further

    The more users use open source presentations based on web frameworks, the higher the chances that developers in open source communities will maintain and update frameworks.

EFFICIENCY

  • Fonts, stylesheets and transitions are pre-rendered for collaborative efficiency

    If there is one thing I have to swear by it is the efficiency of web based open source presentations. Fonts, styling and transitions are pre-rendered which reduces the need for content creators and designers to struggle with the less rewarding aspects of presentations.

  • Developer satisfaction in writing broad multi-purpose functionality

    Developers also focus on modern programming challenges rather than trying to find hacks for proprietary software. Carefully selected web hosting plans can ensure that web based presentations are allocated only the percentage of computing power that they need with options to scale up as per requirement (elasticity)

  • Reduces need for single-use hardware

    There is no need to buy hardware for single-use purposes, for example purchasing a pen drive just to store one or two presentations

  • No need to purchase a computer system just to run a particular software

    Computer users need not give consideration to whether a particular computer model can run the 'essentials' such as the MS Office suite.

  • Non-developers have access to tools

    If you are not a developer you can use https://slides.com to create presentations, which reduces dependency on proprietary software.

  • No need to worry about file formats

    Standard html extension reduces the stress of figuring out whether the file is stored in the correct format.

RESPONSIVENESS

  • Full viewing experience on mobile phones

    Yes your users can get the full viewing experience on mobile phones

  • Stack content correctly

    Media queries can be used to stack content correctly so that floating elements are arranged in full-width blocks

  • Font size can also be increased as per device requirements

    Font size can also be increased as per device requirements

  • Scale down slow loading features

    Heavy animations or transitions can be removed or gradually scaled down to remove the load on mobile devices

  • Scale images appropriately

    Images can be appropriately scaled so that they do not lose their visual impact

ATTRACTIVENESS

  • Images can easily be controlled

    Code can be added so that high resolution full-screen options can be tailored for desktops or mobile forms with retina display via media queries

  • Fonts can easily be styled and managed
  • Less chance of unattractive gaps and misalignment
  • Tweak existing web templates

    Existing web templates which outnumber Powerpoint templates can be tweaked to add presentation functionality.

  • More depth and gradients possible
  • More flexibility for animations and gifs.

    In fact SVGs can be used to create seamless user experience using transitions as a starting point, rather than an afterthought

THE CHALLENGES

  • Many of us were taught and trained in the use of proprietary sofware

    This includes educational institutes and offices. Consequently there is more support and awareness around these technologies.

  • Non-coders may face disadvantages It must be pointed out that training staff members to modify or write in Markup may be easier than training them in proprietary software. There are also WYSIWG tools like https://slides.com to help non-coders.
  • Proprietary technology has first-mover advantage

    MS Powerpoint has actually achieved a complex level of functionality in the form of features over the years and catching up may be more difficult than one would assume.

THE NEED

  • An emerging tool for capacity development particularly for all businesses
  • Challenging developersto commit to innovative projects

    Today, talented developers do not what to hack proprietary software for single-purpose functionality. They can be encourages to build libraries of transition effects or add functionality that will be used by multiple organisations is a more rewarding use of their time and abilities

  • There is a diversion of labour into unnecessary and tedious tasks such as formatting.
  • Dissatisfaction of being associated with 'middlemen' jobs

    There is also the dissatisfaction of being associated with 'middlemen' jobs such as having to merely replicate style and content. Web frameworks can be created side-by-side with web application prototypes, newsletters or social media campaigns. The creative team can tailor different styles and organise content at one go because of the reusability and responsiveness granted by web based open source presentations.

  • Time and energy is wasted on tracking down changes as presentations are typically sent back and forth.
